# Полное описание установленных библиотек

Ниже приведён подробный разбор **всех зависимостей**, которые я установил для проекта.  
Файл создан так, чтобы я мог **всегда быстро открыть, перечитать и понять, что за что отвечает**.

---

# 1. Основная зависимость

## **express**
Express — это минималистичный и гибкий веб‑фреймворк для Node.js.

### Зачем нужен:
- создание HTTP‑серверов
- роутинг (GET, POST, PUT, DELETE)
- middleware (аутентификация, логирование, валидация)
- основа для REST API и backend‑логики

---

# 2. TypeScript и типы

## **typescript**
TypeScript — надмножество JavaScript с типизацией.

### Зачем нужен:
- строгие типы
- лучшее автодополнение
- меньше ошибок на ранних этапах
- более чистая архитектура кода

---

## **@types/express**
Типы Express для TypeScript.

### Зачем нужны:
- чтобы Express корректно работал с TS
- чтобы были типы Request, Response, NextFunction
- IntelliSense понимал структуру express-приложения

---

## **@types/node**
Типы Node.js.

### Зачем нужны:
- чтобы TS понимал глобальные объекты Node (`process`, `__dirname`, Buffer)
- чтобы можно было использовать системные модули (`fs`, `path`, `http`)

---

# 3. Тестирование

## **jest**
Jest — тестовый фреймворк.

### Зачем нужен:
- написание unit-тестов
- написание интеграционных тестов
- создание моков, стабов
- быстрый запуск тестов

---

## **ts-jest**
Преобразователь TypeScript → Jest.

### Зачем нужен:
- позволяет Jest запускать тесты, написанные на TS
- позволяет не собирать проект перед тестированием
- полностью интегрируется с tsconfig

---

## **@types/jest**
Типы для Jest.

### Зачем нужны:
- чтобы TypeScript понимал функции Jest (`describe`, `it`, `expect`)
- корректно работал IntelliSense

---

## **supertest**
Библиотека для e2e-тестирования HTTP.

### Зачем нужен:
- тестирование API без запуска приложения на реальном порту
- отправка запросов напрямую в express-приложение
- удобное тестирование маршрутов, статусов, JSON-ответов

---

## **@types/supertest**
Типы для Supertest.

---

# 4. Линтинг и форматирование

## **eslint**
ESLint — инструмент анализа качества кода.

### Зачем нужен:
- обнаруживает ошибки в коде
- находит проблемные конструкции
- улучшает читаемость
- обеспечивает единый стиль кода в проекте

---

## **@typescript-eslint/eslint-plugin**
Набор правил ESLint для TypeScript.

### Зачем нужен:
- делает ESLint совместимым с TS
- добавляет десятки правил для TS-проектов
- улучшает качество типов

---

## **@typescript-eslint/parser**
Парсер, позволяющий ESLint понимать TypeScript.

### Зачем нужен:
- без него ESLint видит TS как ошибки
- обязательный компонент для lint TS-проектов

---

## **prettier**
Prettier — форматировщик.

### Зачем нужен:
- приводит код к единому стилю автоматически
- устраняет споры о стилях
- экономит время

---

## **eslint-config-prettier**
Отключает правила ESLint, конфликтующие с Prettier.

### Зачем нужен:
- чтобы ESLint и Prettier работали вместе
- чтобы не было двойных ошибок о формате

---

## **eslint-plugin-prettier**
Запускает Prettier внутри ESLint.

### Зачем нужен:
- форматирование запускается через `npm run lint`
- не нужно отдельно запускать Prettier

---

# 5. Утилиты для разработки

## **nodemon**
Утилита, которая перезапускает Node.js приложение при изменении файлов.

### Зачем нужен:
- автоматический перезапуск сервера
- удобная и быстрая разработка
- не нужно вручную писать: `node dist/index.js`
