#  Скрипты package.json — Полное объяснение

Этот документ подробно объясняет все скрипты, которые добавлены в `package.json`, зачем они нужны, и как они используются в ежедневной разработке.

Рекомендуемая структура для хранения документации:

```
docs/
 └── scripts/
      └── package-scripts.md
```

---

#  Описание скриптов

##  1. watch
```
"watch": "tsc -w"
```

### Что делает:
- запускает TypeScript-компилятор в режиме наблюдения
- следит за изменениями в `src/`
- автоматически пересобирает `.ts` → `.js` в `dist/`

---

##  2. dev
```
"dev": "nodemon --inspect dist/index.js"
```

### Что делает:
- запускает приложение из `dist/index.js`
- включает Node.js Inspector
- перезапускает сервер при изменениях

---

##  3. jest
```
"jest": "jest -i"
```

### Что делает:
- запускает Jest в однопоточном режиме
- повышает стабильность тестов

---

##  4. lint
```
"lint": "eslint --fix ."
```

### Что делает:
- запускает ESLint по всему проекту
- исправляет проблемы автоматически

---

##  5. format
```
"format": "prettier --write ."
```

### Что делает:
- форматирует весь проект по правилам Prettier

---

#  Итог

| Скрипт | Для чего нужен |
|--------|----------------|
| watch | Авто-компиляция TypeScript |
| dev | Запуск сервера с авто-перезапуском |
| jest | Запуск тестов |
| lint | Линтинг и автоисправления |
| format | Форматирование проекта |
